# üöÅ Fighter Pilot Bash - Render Deployment Guide

This guide will help you deploy the Fighter Pilot Birthday Bash RSVP application to **Render** (all-in-one platform for frontend, backend, and database).

## Quick Deployment

### 1. Fork & Connect Repository
1. Fork this repository to your GitHub account
2. Go to [render.com](https://render.com) and sign up/login
3. Connect your GitHub account to Render

### 2. Deploy with Blueprint
1. In Render dashboard, click "New" ‚Üí "Blueprint"
2. Connect this repository
3. Render will automatically detect the `render.yaml` configuration
4. Click "Deploy" and wait for the build to complete

That's it! Your application will be deployed with:
- **Web Service**: Full-stack app (frontend + backend)
- **PostgreSQL Database**: Managed database with automatic backups
- **Environment Variables**: Auto-configured for production

## Manual Setup (Alternative)

If you prefer manual configuration:

### 1. Create PostgreSQL Database
1. In Render dashboard: "New" ‚Üí "PostgreSQL"
2. Name: `fighter-pilot-db`
3. Plan: Free
4. Click "Create Database"

### 2. Create Web Service
1. In Render dashboard: "New" ‚Üí "Web Service"
2. Connect your GitHub repository
3. Configure:
   - **Build Command**: `npm ci && npm run build`
   - **Start Command**: `npm run start`
   - **Environment Variables**:
     - `NODE_ENV`: `production`
     - `DATABASE_URL`: (Link to your PostgreSQL database)
     - `SESSION_SECRET`: (Generate a random 32-character string)
     - `ADMIN_PASSWORD`: (Set a strong admin password)

## Environment Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `DATABASE_URL` | PostgreSQL connection string | Auto-generated by Render |
| `SESSION_SECRET` | Session encryption key | `openssl rand -base64 32` |
| `ADMIN_PASSWORD` | Admin panel password | `your-secure-password` |
| `NODE_ENV` | Environment mode | `production` |

## Optional: SMS Verification

To enable SMS verification with Twilio, add these variables:

| Variable | Description |
|----------|-------------|
| `TWILIO_ACCOUNT_SID` | Your Twilio Account SID |
| `TWILIO_AUTH_TOKEN` | Your Twilio Auth Token |
| `TWILIO_VERIFY_SERVICE_SID` | Your Twilio Verify Service SID |

## Database Management

### Push Schema
```bash
# Set your production database URL
export DATABASE_URL="your-render-database-url"

# Push the schema
npm run db:push
```

### Seed Initial Data
```bash
npm run db:seed
```

## üö® Troubleshooting

### Build Failures
- Check build logs in Render dashboard
- Ensure all dependencies are in `package.json`
- Verify build command is correct

### Database Connection Issues
- Verify `DATABASE_URL` is set correctly
- Check database service is running
- Ensure web service has access to database

### Session Issues
- Verify `SESSION_SECRET` is set
- Check PostgreSQL session store is working
- Clear browser cache if needed
